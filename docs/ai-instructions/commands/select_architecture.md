# Select Architecture

| Field | Value |
|-------|-------|
| `name` | `select_architecture` |
| `phase` | project_initiation |
| `intent` | decide |
| `ai` | `required` |

---

## Rules

- Present at least two candidate architecture patterns relevant to the application type (e.g. monolith, microservices, event-driven, serverless)
- Evaluate each candidate against the application requirements and methodology
- Get explicit team confirmation of the chosen architecture
- Record the decision with rationale, trade-offs considered, and any constraints in architecture-decision.md
- Do not proceed to system design without a recorded architecture decision
